📖 Glossário Técnico

Entenda os conceitos fundamentais para integrar com a nossa API.

O JWT é um padrão para transmissão segura de informações entre partes como um objeto JSON. Na nossa API, ele é o "crachá" que você usa após o login.

  • Stateless: O servidor não precisa guardar sua sessão, o próprio token carrega seus dados.
  • Como usar: Envie o token em todas as requisições no cabeçalho Authorization com o prefixo Bearer.
Authorization: Bearer eyJhbGciOiJIUzI1Ni...

É o método usado apenas na rota de Login para obter seu token JWT. Ele consiste em codificar seu usuário e senha em Base64.

Como criar o token Basic:

  1. Junte seu usuário e senha com dois pontos: usuario:senha
  2. Codifique essa string para Base64.
  3. Envie no cabeçalho Authorization: Basic [TOKEN_BASE64]

Exemplo (JS): btoa('meu_usuario:minha_senha')

Para garantir a estabilidade do servidor Firebird, limitamos a frequência de requisições.

  • Limite por Minuto: Evita picos de tráfego que podem travar o banco.
  • Limite Mensal: Controla a cota de uso conforme o seu plano contratado.
  • Excedeu? A API retornará o status 429 Too Many Requests.

RBAC: Role Based Access Control. Significa que seu acesso é baseado em permissões específicas atreladas ao seu usuário.

Endpoints: São os endereços da API (ex: /products, /orders). Cada um representa um recurso do ERP.
